# CH04. 의존 자동 주입 (@Autowired, @Qualifier)

---

## 이 장의 핵심 한 줄 요약

> **CH03에서는 @Bean 메서드 안에서 직접 연결했다면, CH04에서는 @Autowired를 붙이면 스프링이 알아서 찾아 넣어준다.**

---

## 하고 싶은 말이 뭔데?

### CH03 방식 (수동 연결)
```java
@Bean
public MemberRegisterService memberRegisterService() {
    return new MemberRegisterService(memberDao());  // 내가 직접 넣어줌
}
```

### CH04 방식 (자동 주입)
```java
public class MemberRegisterService {
    @Autowired
    private MemberDao memberDao;   // 스프링이 알아서 넣어줌!
}
```

**"@Autowired 붙이면 끝?"** -> 맞다. 스프링 컨테이너가 타입이 맞는 Bean을 찾아서 자동으로 넣어준다. 하지만 같은 타입의 Bean이 2개 이상이면? -> `@Qualifier`로 구분한다.

---

## 반드시 알아야 할 핵심 개념

### 1. @Autowired - 자동 주입

스프링이 **타입을 기준으로** 알맞은 Bean을 찾아 자동 주입한다.

3가지 위치에 붙일 수 있다:

#### (1) 필드 주입
```java
public class MemberRegisterService {
    @Autowired
    private MemberDao memberDao;    // 필드에 직접
}
```

#### (2) setter 주입
```java
public class ChangePasswordService {
    private MemberDao memberDao;

    @Autowired
    public void setMemberDao(MemberDao memberDao) {
        this.memberDao = memberDao;    // setter에
    }
}
```

#### (3) 생성자 주입
```java
public class MemberRegisterService {
    private MemberDao memberDao;

    @Autowired
    public MemberRegisterService(MemberDao memberDao) {
        this.memberDao = memberDao;    // 생성자에
    }
}
```

### 2. @Qualifier - 같은 타입 Bean이 여러 개일 때

**문제 상황:**
```java
@Bean
public MemberPrinter memberPrinter1() { return new MemberPrinter(); }

@Bean
public MemberSummaryPrinter memberPrinter2() { return new MemberSummaryPrinter(); }
// MemberSummaryPrinter는 MemberPrinter를 상속!
// -> MemberPrinter 타입 Bean이 2개 = @Autowired만으로는 어느 걸 넣을지 모름
```

**해결: @Qualifier로 이름표 붙이기**

```java
// 설정 클래스에서 이름표 붙이기
@Bean
@Qualifier("memberPrinter")
public MemberPrinter memberPrinter1() { return new MemberPrinter(); }

@Bean
@Qualifier("summaryPrinter")
public MemberSummaryPrinter memberPrinter2() { return new MemberSummaryPrinter(); }
```

```java
// 주입받는 쪽에서 이름표로 지정
public class MemberListPrinter {
    @Autowired
    @Qualifier("summaryPrinter")          // "summaryPrinter" 이름표 붙은 Bean을 넣어줘
    private MemberPrinter memberPrinter;
}

public class MemberInfoPrinter {
    @Autowired
    @Qualifier("memberPrinter")           // "memberPrinter" 이름표 붙은 Bean을 넣어줘
    public void setMemberPrinter(MemberPrinter memberPrinter) {
        this.memberPrinter = memberPrinter;
    }
}
```

### 3. @Autowired(required = false) - 선택적 주입

**해당 타입의 Bean이 없어도 에러 안 남!**

```java
public class MemberPrinter {
    private DateTimeFormatter dateTimeFormatter;

    @Autowired(required = false)    // Bean이 없으면 이 메서드 자체가 호출 안 됨
    public void setDateTimeFormatter(DateTimeFormatter dateTimeFormatter) {
        this.dateTimeFormatter = dateTimeFormatter;
    }

    public void print(Member member) {
        if (dateTimeFormatter == null) {
            // 포맷터 없으면 기본 형식으로 출력
            System.out.printf("... 등록일 = %tF\n", member.getRegisterDateTime());
        } else {
            // 포맷터 있으면 커스텀 형식으로 출력
            System.out.printf("... 등록일 = %s\n",
                dateTimeFormatter.format(member.getRegisterDateTime()));
        }
    }
}
```

| 옵션 | Bean이 없을 때 |
|---|---|
| `@Autowired` (기본) | **에러 발생** (NoSuchBeanDefinitionException) |
| `@Autowired(required = false)` | 메서드 자체를 호출하지 않음 (필드는 null) |

### 4. 자동 주입 vs 수동 주입 - 우선순위

**@Bean에서 수동으로 넣어줬는데, 필드에 @Autowired도 있으면?**

```java
// 설정 클래스
@Bean
public MemberInfoPrinter memberInfoPrinter() {
    MemberInfoPrinter printer = new MemberInfoPrinter();
    printer.setMemberPrinter(summaryPrinter);  // 수동으로 summaryPrinter 넣음
    return printer;
}

// MemberInfoPrinter 클래스
public class MemberInfoPrinter {
    @Autowired
    @Qualifier("memberPrinter")    // 자동으로 memberPrinter 넣음
    public void setMemberPrinter(MemberPrinter p) { ... }
}
```

-> **@Autowired가 이긴다!** 자동 주입이 수동 주입보다 나중에 실행되므로 덮어씀.

---

## 한눈에 정리

```
@Autowired 동작 흐름:

1. 스프링 컨테이너에서 해당 타입의 Bean을 검색
2. Bean이 1개 -> 그대로 주입
3. Bean이 0개 -> required=true면 에러, false면 무시
4. Bean이 2개 이상 -> @Qualifier로 구분
   - @Qualifier 있으면 -> 이름 매칭
   - @Qualifier 없으면 -> 에러 (NoUniqueBeanDefinitionException)
```

---

## Spring Boot와 뭐가 다른가 / 생소한 부분

| Spring Boot에서는 | 이 챕터에서는 |
|---|---|
| `@Autowired` 자주 씀 (동일) | 동일! 이건 Spring Boot에서도 똑같이 사용 |
| 생성자 주입이면 `@Autowired` 생략 가능 | 순수 Spring 4.3+도 생성자 1개면 생략 가능 (예제에서는 명시적으로 붙여서 보여줌) |
| `@Primary`로도 우선순위 지정 | `@Qualifier`만 사용 |
| `@Component` + `@Autowired` 조합 | `@Bean` 수동 등록 + `@Autowired` 자동 주입 혼합 |

**생소할 수 있는 부분:**
- `@Qualifier`를 @Bean에도 붙이고 주입받는 쪽에도 붙여야 한다는 것
- `required = false`와 `null 체크`의 관계
- 수동 주입과 자동 주입의 우선순위 (자동이 이김)

## 시험 출제 포인트

### 객관식/단답형

1. **@Autowired는 무엇을 기준으로 Bean을 찾는가?** -> 타입(Type)
2. **같은 타입의 Bean이 2개 있을 때 해결 방법은?** -> @Qualifier
3. **@Autowired(required = false)일 때 Bean이 없으면?** -> 해당 setter 메서드가 호출되지 않음 (필드는 null 유지)
4. **@Autowired를 붙일 수 있는 위치 3가지는?** -> 필드, setter 메서드, 생성자
5. **수동 주입(@Bean 메서드 내)과 자동 주입(@Autowired) 중 우선순위가 높은 것은?** -> @Autowired (자동 주입)

### 서술형

1. **@Autowired와 @Qualifier의 관계를 코드 예시와 함께 설명하시오**

   MemberPrinter를 상속한 MemberSummaryPrinter가 있어서 MemberPrinter 타입 Bean이 2개일 때, @Autowired만으로는 어떤 Bean을 주입할지 모르므로 에러 발생. @Qualifier("이름")을 @Bean 정의부와 주입부 양쪽에 붙여서 구분.

2. **필드 주입, setter 주입, 생성자 주입의 차이점과 각각의 장단점을 설명하시오**
   - 필드 주입: 코드가 간결하지만 테스트 시 의존 객체 교체가 어려움
   - setter 주입: 선택적 의존(optional)에 적합, 객체 생성 후 주입
   - 생성자 주입: 불변 보장, 필수 의존성 강제, 테스트 용이 (가장 권장)

---

## 이해도 체크리스트

- [ ] @Autowired가 타입 기반으로 Bean을 찾는다는 걸 안다
- [ ] 같은 타입 Bean이 2개일 때 왜 에러가 나는지, @Qualifier로 어떻게 해결하는지 안다
- [ ] @Qualifier를 @Bean 쪽과 @Autowired 쪽 양쪽에 붙여야 한다는 걸 안다
- [ ] required = false가 뭔지, Bean이 없을 때 어떻게 동작하는지 안다
- [ ] 필드/setter/생성자 주입의 차이를 설명할 수 있다
- [ ] 자동 주입이 수동 주입보다 우선한다는 걸 안다

---

## 검토 보강 포인트

- 실제 코드에서는 `memberPrinter1`, `memberPrinter2`라는 **Bean 메서드 이름**과 `memberPrinter`, `summaryPrinter`라는 **Qualifier 값**이 다르다. 즉, `@Qualifier`는 단순히 Bean 이름 복붙이 아니라 **주입용 식별자**로 이해하면 된다.
- `@Autowired`는 **스프링이 관리하는 객체(Bean)** 에만 동작한다. 직접 `new`로 만든 객체에는 자동 주입이 적용되지 않는다.
- 순수 스프링에서도 **생성자가 하나뿐이면 `@Autowired`를 생략할 수 있다**. 다만 현재 학습용 예제는 자동 주입이 일어나는 지점을 분명히 보여주려고 명시적으로 붙여둔 것으로 보면 된다.
- 예제에는 필드 주입과 setter 주입이 많이 나오지만, 실무나 테스트 관점에서는 **생성자 주입이 가장 권장**된다. CH04는 "자동 주입이 어디에 붙을 수 있나"를 보여주기 위한 장이라고 보면 된다.
- `@Bean` 메서드에서 수동으로 setter를 호출해 넣은 값이 나중에 `@Autowired`로 덮이는 이유는, **Bean 생성 후 자동 주입 후처리(BeanPostProcessor 계열)** 가 뒤에서 한 번 더 돌기 때문이다.
- 선택적 주입은 `@Autowired(required = false)` 외에도 `@Nullable`, `Optional<T>` 같은 방식으로 표현할 수 있다는 점도 알아두면 좋다.
